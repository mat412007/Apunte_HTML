    Información general de HTML.

  • HTML es un lenguaje de programación con el que se hacen las páginas web y aplicaciones.

  • HTML significa Hyper Text Markup Languaje.

    • Hyper Text significa texto dentro de texto, un link por ejemplo es un hypertext.

    • Markup Language significa que el texto es variado y dinámico, con imágenes, links, y muchas otras cosas.

  • Web Page es un documento escrito en HTML y traducido por un navegador web.

  • HTML está compuesto por etiquetas, y cada etiqueta tiene contenido.

  • La estructura inicial o básica de un documento HTML se compone de:

    • <!DOCTYPE> define el tipo de documento y la versión del HTML usado.

    • <html> contiene cada elemento del HTML excepto !DOCTYPE, describe la página web.

    • <head> contiene info. que describe el documento, pero que no aparece con él.

    • <title> título de la página que aparece en el navegador, se inserta en head.

    • <body> su contenido es visible para el usuario, contenido principal de HTML.

    • <h1> primer nivel de heading de la página, son 6 niveles.

    • <p> su contenido es el de un párrafo de la página.

  • Tim Berners-Lee creó HTML, y lo lanzó en 1991.

  • La versión más nueva de HTML es HTML5, lanzada en 2008.

  • Características de HTML:

    • Un lenguaje simple y fácil.

    • Provee variedad de presentación y diseño.

    • Puede usarse en cualquier plataforma.

    • Permite insertar links, imágenes, sonidos y gráficos.

  • Al guardar un archivo HTML se le agrega la extensión .html.

  • Un documento HTML consiste en etiquetas y atributos.

    • Una etiqueta rodea el contenido que se le ponga y le da significado.

    • Un atributo provee información extra al elemento en donde se inserte.

  • Un elemento es el componente del archivo HTML, puede haber infinitos, es el conjunto de una etiqueta y su contenido.

  • Estructura de un elemento:

    • <p class="">This is the content<p/>

  • Algunas etiquetas no necesitan etiquetas de cierre.

  • <br> significa break line, actua como la tecla Enter, inicia una nueva linea.

  • <hr> significa horizontal rule, traza una línea horizontalmente.

  • <style> y style="" puede usarse como elemento y atributo, se usa para insertar CSS en HTML directamente.

  • title="" muestra una descripción del contenido cuando el cursor está sobre él. CPU

  • Tanto comillas simples como dobles sirven para escribir atributos(' ' y " ")

  • Elementos sin etiqueta de cierre se llaman elementos vacios.

  • Un elemento puede contener otro elemento.

  • Por defecto, los elementos HTML se dividen en:

    • Blockl-level element: son bloques que abarcan todo el ancho de su alto, y pueden contener ambos tipos de elementos.

    • Inline element: abarcan la sección de su texto, nada más, no pueden contener bloques, pero sí líneas.

  • <div> crea secciones y agrupa contenidos, es un elemento en bloque.

  • <span> contiene frases en línea para darles estilos.

  • En HTML hay etiquetas que se usan para dar estilo al texto:

    • Etiquetas físicas: otorgan apariencia visual al texto.

    • Etiquetas lógicas: añaden valor lógico o semántico al texto.

  • Algunas etiquetas de formateo son:

    • <b> convierte el texto a negrita. Texto

    • <i> convierte el texto a cursiva. Texto

    • <u> subraya el texto. Texto

    • <sup> despliega texto arriba de la línea. 22

    • <sub> despliega texto debajo de la línea. H2O

    • <del> despliega texto tachado. Texto

  • Los HTML headings son títulos y subtítulos, de mayor a menor.
    • h1, h2, h3, h4, h5, h6.

  • Las etiquetas de frase que definen la estructura de un bloque de texto y sus semánticas son:

    • <em> hace lo mismo que <i>, pero con extra semánticas. Texto

    • <mark> destaca una parte importante del texto marcandoo con amarillo. Texto

    • <strong> remarca texto como <b>, pero le da importancia a la máquina también. Texto

    • <abbr> se usa para abreviar texto. HTML

    • <dfn> pone texto a definir en cursiva. Definición

    • <code> se usa para poner código de computadora en formato de texto. <p>Texto<p/>

    • <kbd> se usa para representar teclas del Keyboard. Enter + C

    • <address> etiqueta de bloque que representa info. del autor de la página.

    Un hipervínculo en HTML.

  • <a> crea un hyperlink a otra página, carpeta o URL.

    • href="" es el atributo más importante de <a>, contiene el link de la página o URL a la que desea llegar.

    • target="_blank" es un atributo que se usa para que el link se abra en una nueva pestaña.

    Una imagen en HTML.

  • <img> despliega una imagen en la página web, no tiene contenido o etiqueta de cierre.

    • src="" contiene la fuente o camino de la imagen, nos dice donde buscarla.

    • alt="" contiene texto alternativo que aparece en el lugar de la imagen en caso de no ser desplegada.

    • width="" y height="" definen el ancho y el alto de la imagen, y ambos se usan en CSS.


    Tablas de HTML.

  • Una tabla en HTML se compone con muchas etiquetas y atributos:

    • <table> abre y cierra el cuadro de la imagen, en su interior están las etiquetas que definen la tabla.

    • <tr> define una fila(row) de la tabla.

    • <th> define el titulo(header) de la columna.

    • <td> define el contenido(data) de cada espacio de la tabla.

    • <caption> título de la parte de la tabla.

    • <colgroup> especifica un grupo de columnas para personalizarlas.

    • <col> se usa en <colgroup> para especificar la columna.

    • <thead> agrupa todos los <th>.

    • <tbody> agrupa el contenido general de la tabla, excepto la superior.

    • border="" especifica el borde de la tabla.

    • padding="" especifica el relleno de los <th> y <td>.

    • width="" especifica el ancho de la en % o px.

    • height=""especifica el alto del <table>, y solo puede ponerse en px.

    • border-collapse: collapse; colapsa los bordes en uno solo.

    • colspan="" se usa para que una columna tenga dos espacios para escribir.

    • rowspan="" se usa para que una fila tenga dos espacios para escribir.

    Listas de HTML.

  • Hay varios tipos de listas en HTML:

    1. <ol> produce una lista ordenada por números o letras.

    • <ul> produce una lista desordena cuyos items están marcados por símbolos como círculos o cuadrados.
    <dl>
    Produce una lista de definiciones.

    <dt>
    Definition term, el término que se va a definir.

    <dd>
    Definition data, información del término que se define.

  • Una lista ordenada puede tener letras o números, y estar en cualquier orden.

    1. type="1" ordena la lista con números.

    1. type="I" ordena la lista con números romanos en mayúscula.

    1. type="i" ordena la lista con números romanos en minúscula.

    1. type="A" ordena la lista con letras mayúsculas.

    1. type="a" ordena la lista con letras minúsculas.

    1. start="5" comienza la lista con el valor que le insertes.

    • reversed voltea el orden de la lista.

  • Una lista no ordenada se ordena con símbolos:

    • type="circle" ordena la lista con circulos.

    • type="square" ordena la lista con cuadrados.

    • type="disc" ordena la lista al igual que su estilo predeterminado.

    • type="none" la lista no tiene marcas.

    Formularios de HTML.

  • <form> marca una sección del documento que contiene controles con los que el usuario puede interactuar, como contraseñas, botones, preguntas de opción múltiple, y muchas otras.

    • <input> se inserta dentro de form, define un control de input, una línea de texto.

      • <label> representa una etiqueta para los input, describe la información que se debe insertar.

      • value="" le asigna un valor, propiedad o función diferente al <input> según el tipo que sea.

      • id="" atributo de <input> para conectarlo con su <label>, el label tiene el for="", y ambos deben ser iguales.

      • for="" atributo de <label> para conectarlo con su <input>, el input tiene el id="", y ambos deben ser iguales.

      • placeholder="" texto que aparece en el input antes de insertar algo en él.



    • <textarea> define un control input de más de una línea de texto.

      • rows="" define la cantidad de filas.

      • cols="" define la cantidad de columnas.


    • <fieldset> agrupa varios campos de control, y los rodea con un borde predeterminado.

      • <legend> es el caption del fieldset, algo como un minitítulo.

    • <select> comienza y termina una lista de dropdown.

      • <optgroup> define un grupo de opciones relacionadas en un dropdown list.

      • <option> define una opción en un dropdown list.


    • <button> define un botón clickeable.


  • El atributo type="" de <input> puede ser de varios tipos:

    • type="text" define un <input> de texto.

    • type="password" define un <input> para ingresar una contraseña.

    • type="submit" define un botón para entregar el <form>.

    • type="reset" define un botón para reinicar el <form>.

    • type="radio" define un botón de radio para seleccionar solo una opción correcta.

    • type="checkbox" define una caja para seleccionar más de una opción.

    • type="button" define un botón cuya función puede ser programada.

    • type="file" un input para seleccionar una carpeta de almacenamiento.

    • type="image" define un botón con la apariencia de una imagen, al presionarse envía el <form> al que pertenezca.

    • type="color" define un color con un color específico.

    • type="date" define un <input> para insertar una fecha.

    • type="datetime-local" define un <input> para ingresar una fecha local.

    • type="email" define un <input> para un email.

    • type="month" define un <input> para inf¡gresar un mes.

    • type="number" define un <input> para un número.

    • type="url" define un <input> para un URL.

    • type="week" define un <input> para seleccionar una semana del año.

    • type="search" define un <input> para búsqueda.

    • type="tel" define un <input> para ingresar un número de telefono.

  • El elemento HTML <form> tiene varios atributos:

    • action="" define a donde vas a enviar el <form>, ya sea un mail o una página web.

    • method="" define el método por el cual se envía el formulario. Sus valores pueden ser:

      • method="get" envía los datos y la URL del formulario, cualqiera es capaz de verlo, por lo que se considera poco seguro.

      • method="post" envía los datos del formulario pero no la URL, no son visibles para nadie excepto a quien se lo envíes, es un método muy seguro.

    • target="" define donde abrir la respuesta de que se envió el formulario. Hay 2 opciones principales:

      • target="_self" abre la respuesta en la página actual.

      • target="_blank" abre la respuesta en una nueva pestaña.

    • autocomplete="" define si el form puede completarse solo, sus 2 posibilidades son on y off.

    • enctype="" define el tipo de codificación del <form> al entregarse.

      • application/x-www-form-urlencoded codifica los caracteres antes de la entrega.

      • multipart/form-data se usa al tener que subir carpetas.

      • text/plain los caracteres no se modifican, excepto los espacios reemplazandolos por +.

    • novalidate el <form> no se valida al entregarse.

  • El elemento <input> tiene muchos atributos también:

    • name="" define el nombre del <input>, representa su valor al momento de la entrega.

    • value="" define el valor del <input>.

    • requiered especifica que el <input> debe ser completado antes de entregarse.
